Project 13: Interview Preparation Chatbot with Real-Time Feedback 
Day 1: Project Document & Architecture Setup (20 Marks) 
Focus Area: Problem & Planning 
●​ Technical Tasks: Define the multi-modal architecture. Map out the client-side (React + 
MediaPipe + WebRTC) vs. server-side (FastAPI + Whisper + GPT-4o) responsibilities. 
Design the PostgreSQL schema for user profiles, session logs, and scorecards. 
●​ Deliverables: A formal Project Document with the problem statement, feature list, 
system architecture diagram, and assigned team roles. 
●​ How to ace the KPIs: 
○​ Clarity of problem (5): Explicitly state that students fail technical interviews not 
because they lack knowledge, but because they lack personalized, high-fidelity 
behavioral feedback. 
○​ Feasible scope (5): Bound the project carefully. State: "For the MVP, we will 
focus on Behavioral and System Design interviews, tracking Eye Contact and 
Filler Words. Live coding execution is out of scope." 
○​ Architecture understanding (5): Include a flowchart showing: Webcam → 
Browser (MediaPipe CV) + Audio Stream → Backend (Whisper STT) → GPT-4o 
(STAR Method Evaluation) → React Scorecard. 
○​ Team roles defined (5): Assign clear ownership for the Frontend/Computer 
Vision (React/MediaPipe), the Audio/Backend (WebRTC/Whisper), and the LLM 
Evaluation (GPT-4o Prompting). 
Day 2: WebRTC & The Skeleton Pipeline (20 Marks) 
Focus Area: Setup & Basic Pipeline 
●​ Technical Tasks: Scaffold the React video interface. Implement WebRTC to capture the 
user's webcam and microphone. Set up the backend to receive the audio stream, pass it 
to Whisper for transcription, and return the raw text to the frontend. 
●​ Deliverables: A running skeleton where a user speaks into their webcam, and the 
application successfully transcribes their speech into text on the screen in near real-time. 
●​ How to ace the KPIs: 
○​ Code runs (5): The React app successfully asks for camera/mic permissions 
and streams data without crashing. 
○​ API/LLM integration (5): Prove your backend can successfully chunk the audio 
and get accurate transcriptions from the Whisper API. 
○​ Repo + commits (5): All team members have pushed the base React 
components and FastAPI audio handlers. 
○​ Live demo success (5): Speak into the skeleton app live and show the evaluator 
the raw text transcription appearing on the screen. 
Day 3: Computer Vision & Audio Analytics (20 Marks) 
Focus Area: Core Feature 
●​ Technical Tasks: Integrate MediaPipe Holistic into the React frontend. Calculate "Eye 
Contact" by tracking iris position relative to eye corners. Implement the algorithm to 
detect filler words ("um," "like," "you know") and calculate Words Per Minute (WPM) from 
the Whisper transcription timestamps. 
●​ Deliverables: The core tracking intelligence is functional. The app can look at a user 
and calculate their eye-contact percentage and filler-word frequency in real-time. 
●​ How to ace the KPIs: 
○​ Core feature implementation (8): The MediaPipe integration successfully draws 
facial landmarks on the screen (for debugging) and accurately flags when the 
user looks away from the camera. 
○​ Real data usage (4): Test the filler word detection on naturally messy, 
conversational speech, not just perfectly enunciated test phrases. 
○​ Test cases (4): Have specific test cases ready: deliberately look away from the 
camera for 5 seconds and show the eye-contact score dropping dynamically. 
○​ Progress vs plan (4): The client-side CV and server-side audio processing align 
perfectly with the Day 1 architecture. 
Day 4: LLM Evaluation & Feedback Engine (20 Marks) 
Focus Area: Integration & Stability 
●​ Technical Tasks: Send the Whisper transcript + the AI's question to GPT-4o. Write the 
strict system prompt to evaluate the answer based on relevance, completeness, and 
structure (e.g., enforcing the STAR method: Situation, Task, Action, Result). Build the 
logic to generate the comprehensive post-interview scorecard. 
●​ Deliverables: A fully working end-to-end pipeline. The AI asks a question, the user 
answers (while being tracked for CV/Audio metrics), and the system generates a grade 
and feedback. 
●​ How to ace the KPIs: 
○​ End-to-end integration (8): Data flows seamlessly from the initial AI question to 
the user's multi-modal response, culminating in the LLM's structured feedback. 
○​ Stability (no crashes) (4): Handle audio dropouts gracefully. Ensure the LLM 
doesn't crash if the user stays silent for 10 seconds. 
○​ Output quality (4): The GPT-4o feedback is highly specific (e.g., "You explained 
the 'Action' well, but forgot to mention the 'Result' of your project"). 
○​ Debugging effort (4): Keep a log of how you tuned the MediaPipe facial 
landmark thresholds to accommodate users wearing glasses or in low lighting. 
Day 5: Dashboards, Polish & Storytelling (10 Marks) 
Focus Area: Polish & Demo Prep 
●​ Technical Tasks: Finalize the React UI. Build the Post-Interview Scorecard view 
(combining the LLM's technical grade with the CV eye-contact score and Audio WPM 
metrics). Implement subtle real-time on-screen cues (e.g., a small "Slow down" toast if 
WPM exceeds 160). 
●​ Deliverables: A visually polished, professional mock-interview platform and a 
compelling presentation deck. 
●​ How to ace the KPIs: 
○​ UI clarity (3): The video interface feels like Zoom or Google Meet. The 
post-interview scorecard uses clear charts (e.g., radar charts for confidence, 
technical accuracy, and communication). 
○​ Demo readiness (3): The database is pre-loaded with past mock sessions so 
the "Progress Tracking" dashboard shows a realistic improvement curve over 
time. 
○​ Storytelling (2): Frame the project as an equalizer that democratizes access to 
expensive executive interview coaching for university students. 
○​ Feature completeness (2): The WebRTC video, MediaPipe CV, Whisper STT, 
and LLM evaluations are all functioning together. 
Day 6: Final Live Demo & Q&A Defense (10 Marks) 
Focus Area: Final Demo 
●​ Technical Tasks: Execute the live presentation. Ensure your webcam isn't being 
blocked by another application (like Zoom) before you start the live demo. 
●​ Deliverables: A flawless live demo and a strong defense of your technical architecture. 
●​ How to ace the KPIs: 
○​ Demo success (4): Conduct a 1-minute live mock interview. Intentionally use 
filler words and look away from the camera. End the interview and show the 
scorecard correctly docking points for those exact behaviors while grading your 
verbal answer. Have a backup video ready. 
○​ Technical understanding (3): Be highly prepared to explain why you ran 
MediaPipe on the frontend (React/Browser) rather than sending video frames to 
the backend (Answer: to eliminate network latency, ensure real-time feedback, 
and dramatically reduce server compute costs). Evaluators love this architectural 
optimization. 
○​ Team participation (2): Split the demo. One person handles the 
WebRTC/Whisper explanation, another explains the MediaPipe computer vision 
math, and a third drives the live demo and LLM scorecard. 
○​ Q&A handling (1): Be prepared to discuss data privacy (e.g., "We do not store 
the video feeds, we only extract mathematical landmark coordinates in the 
browser, ensuring user privacy").
